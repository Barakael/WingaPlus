# Configure Official `@wingapro.app` Warranty Emails

This project already sends warranty emails through Laravel mail. To move from Gmail to your official domain and improve inbox placement, follow this setup.

## 1) Use a transactional sender (recommended)

For low/irregular volume, use a provider with built-in deliverability reputation:

- Resend
- Postmark
- Mailgun
- Amazon SES

Do not send directly from shared Gmail for production warranty traffic.

## 2) Authenticate your domain DNS

In your DNS provider for `wingapro.app`, add:

- SPF (`TXT` at root): allow your chosen provider
- DKIM (`TXT` records): generated by provider
- DMARC (`TXT` at `_dmarc.wingapro.app`), start with:

```txt
v=DMARC1; p=none; rua=mailto:dmarc@wingapro.app; adkim=s; aspf=s
```

After validating flow, tighten policy (`quarantine` then `reject`).

## 3) Configure app mail env values

Set production environment variables (example with provider SMTP):

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.your-provider.com
MAIL_PORT=587
MAIL_USERNAME=apikey-or-username
MAIL_PASSWORD=your-secret
MAIL_ENCRYPTION=tls

MAIL_FROM_ADDRESS=warranty@wingapro.app
MAIL_FROM_NAME="WingaPro Warranty"

# Optional global fallback for Reply-To
MAIL_REPLY_TO_ADDRESS=support@wingapro.app
MAIL_REPLY_TO_NAME="WingaPro Support"
```

Notes:

- The app now supports a global `MAIL_REPLY_TO_*` fallback.
- Warranty emails also set `Reply-To` dynamically from the issuer shop/user email when available.

## 4) Clear config cache and test

```bash
php artisan config:clear
php artisan cache:clear
php artisan config:cache
php artisan tinker --execute="Mail::raw('WingaPro mail test', function(\$m){ \$m->to('your-email@example.com')->subject('WingaPro Delivery Test'); });"
```

## 5) Warm up domain reputation (important for low volume)

- Start by sending real emails to internal addresses (Gmail/Outlook/Yahoo) and mark as "Not spam".
- Keep content consistent and transactional (warranty confirmation only).
- Avoid link-shorteners and excessive images-only content.
- Keep bounce/complaint rates low; remove invalid addresses quickly.

## 6) Keep sender identity stable

- Always send from one primary address (`warranty@wingapro.app`).
- Use `Reply-To` for shop-specific responses.
- Avoid frequent changes in sender name/address.

## 7) Optional monitoring checklist

- Track bounces, blocks, complaints in provider dashboard.
- Use [Google Postmaster Tools](https://postmaster.google.com/) for domain reputation.
- Periodically test with [mail-tester.com](https://www.mail-tester.com/).

---

If you must temporarily keep Gmail API, set:

```env
MAIL_MAILER=gmail-api
MAIL_FROM_ADDRESS=warranty@wingapro.app
MAIL_FROM_NAME="WingaPro Warranty"
```

But long-term deliverability is better with a domain-authenticated transactional provider.
